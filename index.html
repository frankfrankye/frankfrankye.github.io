MVVM模式和MVC模式是啥

MVC模式是单向绑定，即Model绑定到View，当我们用JavaScript代码更新Model时，View就会自动更新。
MVVM模式就是Model–View–ViewModel模式。用户更新了View，Model的数据也自动被更新了，这种情况就是双向绑定。

通过数据劫持结合发布者-订阅者模式的方式来实现的。一个监听器Observer，一个订阅者Watcher，一个解析器Compile。

this

在浏览器里，在全局范围内this 指向window对象；
在函数中，this永远指向最后调用他的那个对象；
构造函数中，this指向new出来的那个新的对象；
call、apply、bind中的this被强绑定在指定的那个对象上；
箭头函数中this比较特殊,箭头函数this为父作用域的this，不是调用时的this.要知道前四种方式,都是调用时确定,也就是动态的,而箭头函数的this指向是静态的,声明的时候就确定了下来；
apply、call、bind都是js给函数内置的一些API，调用他们可以为函数指定this的执行,同时也可以传参。

原型和原型链

每个函数都有一个prototype属性，被称为显示原型；
每个实例对象都会有_ proto 属性,其被称为隐式原型
每一个实例对象的隐式原型 proto _属性指向自身构造函数的显式原型prototype
每个prototype原型都有一个constructor属性，指向它关联的构造函数。

获取对象属性时，如果对象本身没有这个属性，那就会去他的原型__proto__上去找，如果还查不到，就去找原型的原型，一直找到最顶层(Object.prototype)为止。Object.prototype对象也有__proto__属性值为null。

垃圾回收机制

标记清除：变量被声明后被标记进入环境，离开环境之后再做离开环境标记，收回内存。
引用计数的含义是跟踪记录每个值被引用的次数。当声明了一个变量并将一个引用类型值赋给该变量时，则这个值的引用次数就是1。如果同一个值又被赋给另一个变量，则该值的引用次数加 1。相反，如果包含对这个值引用的变量又取得了另外一个值，则这个值的引用次数减 1。当这个值的引用次数变成 0 时，则说明没有办法再访问这个值了，因而就可以将其占用的内存空间回收回来。这样，当垃圾回收器下次再运行时，它就会释放那些引用次数为 0 的值所占用的内存。

http与https的区别

1、HTTP是超文本传输协议，信息是明文传输，HTTPS是具有安全性的SSL加密传输协议。
2、HTTPS协议需要ca申请证书，一般免费证书少，因而需要一定费用。
3、HTTP和HTTPS使用的是完全不同的连接方式，用的端口也不一样。前者是80，后者是443。
4、HTTP连接是无状态的，HTTPS协议是由SSL+HTTP协议构建的可进行加密传输、身份认证的网络协议，安全性高于HTTP协议。

js中数组排序（冒泡、快速、插入）

冒泡：一个循环嵌套另一个循环 两两比较
插入：将要排序的数组分成两部分，每次从后面的部分取出索引最小的元素插入到前一部分的适当位置
选择：将第一个元素和其他元素作比较，检查完所有的元素后，最小的放在第一个位置，接下来再开始从第二个元素开始，重复以上一直到最后
快速：以一个数为基准(中间的数)，比基准小的放到左边，比基准大的放到右边，而每个小的数组又可以继续看成更小的两个数组，一直递归下去，直到数组长度大小最大为2。
归并：将一个数组反复二分为两个小数组，直到每个数组只有一个元素；其次是“治”，从最小数组开始，两两按大小顺序合并，直到并为原始数组大小

什么是“3次握手，4次挥手

TCP是一种面向连接的单播协议，在发送数据前，通信双方必须在彼此间建立一条连接。所谓的“连接”，其实是客户端和服务器的内存里保存的一份关于对方的信息，如ip地址、端口号等。
TCP可以看成是一种字节流，它会处理IP层或以下的层的丢包、重复以及错误问题。在连接的建立过程中，双方需要交换一些连接的参数。这些参数可以放在TCP头部。
TCP提供了一种可靠、面向连接、字节流、传输层的服务，采用三次握手建立一个连接。采用4次挥手来关闭一个连接。
“3次握手”的作用就是双方都能明确自己和对方的收、发能力是正常的。

实例方法在实际开发中其实用的没这么多，真正用的多的还是以下几种字符串的方法

  • match() 对字符串进行匹配，返回匹配结果，与exec()比较相似，匹配总是从字符串的第一个字符开始
  • search() 返回第一个满足条件的匹配结果在字符串中的位置，如果没有就返回-1
  • replace(a, b) 接受两个值，第一个是要取代内容的位置（可以用search找出来），第二个是替换的内容 ； 取代的是第一个匹配到的值
    replace中的第二个参数还可以是一个函数—— 值得一提的是第二个参数可以用 $ 符号来指代替换的内容
    比如 $n 代表的是匹配成功的第n组内容， n是从1开始的自然数
    其他更多的运用可以自行查看文章底部链接
  • split() 按照正则规则分割字符串，返回一个由分割后的各个部分组成的数组

4.匹配规则

这就是重头戏了

  1. 点字符（.）匹配除回车、换行、行分隔符和段分隔符的所有字符 （/c.t/ 匹配c和t之间任意一个字符）
  2. 位置字符（^）：表示字符串的开始位置 ； ($) ：字符串的结束位置
    e.g /^test/ test必须出现在字符串的开始位置
  3. 选择符（|）

    转义符

    有特殊含义的元字符如果要匹配它们本身的话就需要在前面加上反斜杠
    特殊元字符： ^ . [ $ ( ) | * + ?

    字符类

    表示有一系列字符可供选择，只要匹配其中一个就可以了 【xyz】代表x,y,z中任选一个匹配就返回true
    【^】是脱字符，即除了字符类中字符，其他字符都可以匹配
    【-】 是连字符 【a-z】表示26个字母

    预定义模式

    就是一些预先定义好的简写，可以背一背，也可以无视~
  • \d 相当于【0-9】
  • \D 相当于【^0-9】
  • \w 相当于【0-9a-zA-Z_】
  • \W
  • \s 匹配空格
  • \S

    重复类

    {n}代表重复几次 {n,m}表示重复不少于n次，不多于m次

    量词符

    ？出现0次或1次
  • 出现0次或多次
  • 出现一次或多次

    修饰符

    修饰符一般都放在正则表达式的末尾
    g修饰符 全局匹配
    i 修饰符 忽略大小写
    m修饰符 多行模式，会识别换行符

vuex概念

然后来总结一些vuex的概念：

state、store

在最基本的使用中，只需要定义一个store之后在state里存放状态，
比如说在最新项目中用户登录之后需要设置一个默认的语言，可以在state中添加一个lang属性，当注册之后设置
store.state.lang = navigator.language
这样就可以直接为用户选择浏览器的语言设置为默认语言。

但是如果需要vuex中的多个数据的这时候，这样写就太啰嗦了（其实就算只用这一个也显得好难看），所以可以使用辅助函数。

辅助函数mapState、mapGetters

这中间我其实忽略了直接定义在computed中的那种写法，之所以忽略是因为——

这么写和直接调用完全没啥区别（捂脸
...mapState(['nickname','age','gender'])
...mapGetters(['realname','money_us'])
而如上两个这么写就简洁很多了。
然后忘记说了，mapGetters就相当于计算属性，有些state中的值可以直接通过getters进一步处理。

Mutation、Actions

如上文所说的分属同步与异步的操作。

第一个参数就是state，第二个payload即额外的参数，在这里的例子里就是年龄+5。这两个参数只有mutaition-commit这一组拥有；而actions中的方法默认参数为**”context”**,可以理解为 store 对象。

知乎上就有人问了，直接调用Store.state.age += 5是不是也可以，感觉没什么区别，我觉得也没什么区别，然而大神们告诉我们还是太年轻了——

  1. 在mutations中不仅仅能做赋值操作
  2. 作者在mutations中做了类似埋点操作，如果从mutations中操作的话， 能被检测到，可以更方便用调试工具调试，调试工具可以检测到实时变化，而直接改变state中的属性，则无法实时监测

(二)复杂的vue3和复杂的proxy

Posted on 2022-07-08

proxy

迁移到Vue3的时候，发现了一个非常高频经常出现的内容——proxy。

proxy是啥？

Proxy 可以理解成，在目标对象之前架设一层“拦截”，外界对该对象的访问，都必须先通过这层拦截，因此提供了一种机制，可以对外界的访问进行过滤和改写。
————ECMAScript6 入门

背八股文的时候了解到vue2实现数据双向绑定原理靠的是一个叫Object.defineProperty的东西，他对每一个属性进行了劫持，中间还有一些setter,getter,watch,compiler之类的内容，这些都不重要，因为vue3直接舍弃了这个东西。
我可以先总结一下Object.defineProperty的缺陷，以下都是从网上复制来的：
**无法监听数组基于下标的修改，不支持 Map、Set、WeakMap 和 WeakSet等缺陷 **

好的，我们知道这个东西的缺陷了，大概。

proxy的优缺点和实现

相对于 Object.defineProperty 需要对每一个属性进行劫持，Proxy 更加便捷，只需要劫持需要监听的对象即可。
proxy 的使用本身就是对于 对象的拦截， 通过new Proxy 的返回值，拦截了obj 对象
如此一来，当你 访问对象中的值的时候，他会触发 get 方法， 当你修改对象中的值的时候 他会触发 set方法

你说proxy的缺点？
就使用来说就是每次调用的时候需要写上.value，以及解构的时候会丢失响应。

需要.value的原因？执行ref 的时候，如果是个原始值，他相当于会new 一个对象， 此时你的返回值就是一个对象，这个对象里面包含 一个value 属性 的get 和set 方法，这也是为什么你必须.value的原因
源码：function createRef(rawValue: unknown, shallow: boolean) { if (isRef(rawValue)) { return rawValue } return new RefImpl(rawValue, shallow) }

什么情况下会丢失响应？

  1. 解构 props 对象，因为它会失去响应式
  2. 直接赋值reactive响应式对象
  3. vuex中组合API赋值

(一)SFC和JSX的扩展理解

Posted on 2022-07-04

SFC&JSX

首先，这两个的定义

SFC 就是写 Vue 组件的时候写的.vue文件，这一个文件就是一个 SFC，全称 Single File Component，也即单文件组件。 vue3官网链接

JSX 把HTML代码写在JS里，那就是JSX 把HTML代码写在JS里，那就是JSX

根据Evan You在知乎上的回答，可以给出结论：
————在性能方面，使用模板比使用 TSX 快得多。在同等 IDE 支持的前提下，使用 TSX 本质上是为了开发者的偏好而牺牲了性能（如果没有性能瓶颈也没关系）。

其次，为啥要写JSX

一句话概括：TypeScript 原生支持 JSX 语法，而基本无望 TS 官方能支持 SFC 的 template 语法。
也就是说引入的 SFC 组件，TS 是不知道这个组件的 Props 应该接收什么的。所以无法享受到这些 TS 的优势：

  1. 开发时的自动提示
  2. 编译时的 TS 校验，让你尽早发现问题
  3. 编译组件生成你的组件定义（对于类库开发尤其重要） 所以我要手撸一个组件库的话，基本避不开tsx与jsx

当然你会说既然 Vue 官方能开发处 SFC 的语法，自然会支持这些特性。我表示这当然有可能，但是这个难度是非常大的，需要很多方面的支持，甚至可能需要 TS 官方团队愿意协助，
但是我想不到 TS 官方有什么理由来支持 SFC，因为这只是 Vue 自己创建的方言，在其他场景下是没有使用的，TS 是面向全社区的，我觉得他们不会考虑主动来支持 SFC。

const element = <div id="root">Hello World</div>
这个jsx语法编译过后其实就是：
const element = createElement('div', { id: 'root' }, 'Hello World')

而 JSX 就是这些了，没有什么更多的内容，所以说 JSX 只是方便我们写嵌套的函数调用的语法糖，而其本身没有扩展任何其他的内容。
但是 SFC 就不一样了。
SFC 定义的不仅是语法，更是文件。
SFC 的具体定义是单文件组件，它本身就是把一个文件看作一个单位，所以他的约束性是要大很多的，你必须具有固定的文件结构才能使用 SFC，这做了很多的限制：

  • 一个文件只能写一个组件
  • 节点片段只能写在 template 里面，非常不灵活
  • 变量绑定只能获取this上面的内容，不能使用全局变量（很多时候我们都要把全局变量先挂载到this上）
